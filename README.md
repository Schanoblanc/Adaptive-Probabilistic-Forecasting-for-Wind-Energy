## Data Availability Statement

The data used in this paper were obtained from the Balancing Mechanism Reporting Service (BMRS) API provided by Elexon® (replaced by Insight Solution® in May 2024).

The dataset consists of electricity production records from 128 wind farms in Great Britain from 2020 to 2023  inclusive, with a time resolution of 30 minutes.

The raw data used in this study are included in this reproducibility package and can be found in the `./data/` directory.

The BMRS data included in this package remain subject to the Elexon BSC Open Data Licence. A copy of the licence is provided in the file: `BSC Open Data Licence - Elexon BSC.pdf`.

The following observations are based on our limited experience with the computational environment used in this study and are provided for reference only.
* A machine with at least <mark>**16 GB of RAM**</mark> is recommended. We have not tested the reproducibility code on systems with less memory.
* For disk storage, approximately <mark>**20-30 GB of free disk space**</mark> should be sufficient to run the reproducibility package and store all intermediate files and reproduced outputs.
* High-performance system power profiles may reduce execution time.
* In our experiments, the main model evaluation script was executed using **4 logical CPU cores**, while the sensitivity analysis script was executed using **8 logical CPU cores**. Consequently, the reproducibility package requires a system with at least **8 logical CPU cores**.

The reproducibility code was executed on a <mark>**Windows 11 Pro**</mark> Operating System.

The reproducibility code was executed on a <mark>**CPU**</mark>.

The reproducibility code requires <mark>**parallel processing**</mark>. Therefore, a <mark>**multi-core CPU with at least 8 logical cores**</mark> is required. Systems with fewer than 8 logical cores are not supported.

The reproducibility code requires at least <mark>**8 GB of available RAM**</mark> at runtime. The reproducibility package will terminate if less available memory is detected.

## Expected Discrepancies


### Rounding-Related Discrepancies in CRPS Tables

Minor discrepancies may occur in the CRPS summary tables due to differences in computational environments and the manual rounding process used when preparing the final manuscript tables.

The following files are affected:

- `Table.CRPS_Skill_Appendix.csv`
- `Table.CRPS_Skill_Mainpaper.csv`

The values reported in the manuscript were manually rounded and entered with a precision of three decimal places. Therefore, discrepancies of up to approximately **±0.002%** should be considered acceptable.

For the rank table `Table.Rank.csv`, no discrepancies are expected in principle. However, due to differences in floating-point arithmetic and computational environments, a discrepancy of **±1** in individual rank counts should be considered acceptable.

### Randomness-Related Discrepancies in Sensitivity Analysis

The sensitivity analysis was performed using parallel computation with **8 CPU cores** and random numbers generated by NumPy.
Since fixed random seeds do not necessarily guarantee identical results across different hardware and software environments in parallel computing, no fixed random seed was enforced.
As a result, the reproduced sensitivity-analysis results are expected to differ from the reference results.

Nevertheless, in `Fig.Sensitivity.jpeg`, the following qualitative behaviours should remain consistent:

- For the disturbances on $\mu$, $\sigma_z^2$, and $\nu$, the reproduced results should generally fluctuate around the blue reference values.
- For the disturbance on $M$, the results of **RLS**, **sRLS**, and **RMLE** should generally remain on the left-hand side (lower side) of the blue reference values.
- For the disturbance on $M$, the results of **sRMLE**, **Bayes**, and **Bayes-$\nu$** should generally remain around the blue reference values.
  - For **Bayes** and **Bayes-$\nu$**, the scale of the x-axis should remain smaller than $10^{-5}$.

<h1 id="reproduction-instructions" align="center">Reproduction Instructions</h1>
<!---------------------------------------------------------------------->

In this section, `./` denotes the root directory of the reproducibility package.
When executing scripts, ensure that the current working directory in the terminal is the root directory of the reproducibility package.


## Setup

Before running the reproduction scripts, please complete the following steps:

1. Prepare a suitable machine, virtual machine, or computational environment.
1. Install Python with recommended version.
1. Extract the reproducibility package. We recommend using a directory path that:
   - contains only letters (`a-z`, `A-Z`), numeric characters (`0-9`), and underscores (`_`);
   - contains **NO** spaces; and
   - has a total path length of fewer than **60** characters.
2. Install the required Python packages. The replicator may install them using
    ```powershell
    pip install -r ./src/requirements.txt
    ```
    .


## Configuration Customisation

The only parameter intended to be modified by replicators is `CPU_COUNT` in `./src/Configuration.py`.

Replicators may increase this value according to the number of available CPU cores. This parameter primarily affects the execution time. Minor numerical differences may occur due to differences in parallel execution environments, but no material impact on the reproduced conclusions is expected.

`RELEASE.RunSensitivity` uses a fixed setting of **8 CPU cores**. Replicators who wish to modify this value may edit the following line in `./src/Scripts/RELEASE.RunSensitivity.py` (line 389):

```python
with Pool(processes=8) as pool: results = pool.map(expm, runs)
```

When increasing the number of CPU cores, we recommend doing so in pairs and providing at least **4 GB of available RAM for every 2 CPU cores**.


## Running the Experiments

1. In PowerShell, run:

   ```powershell
   .\RunAll.ps1
   ```

   The `RunAll` script automatically invokes the following scripts in the order:

   - `RELEASE.ConfigCheck`
   - `RELEASE.DataQuality`
   - `RELEASE.RunModels`
   - `RELEASE.RunSensitivity`
   - `RELEASE.FigureTable`

1. `RELEASE.ConfigCheck` performs basic checks on paths, CPU cores, and available RAM. 
   If an error is raised at this stage, please fix it accordingly.
   Alternatively, replicators may disable the checks at their own discretion. We do not recommend bypassing them.

1. Log messages will be printed during execution.
   Replicators may choose to redirect them to a log file or leave them in the terminal.

1. During `RELEASE.RunModels`, it is expected to encounter the warning `ValueWarning: Matrix is singular. Using pinv.`.

1. During `RELEASE.FigureTable`, it is expected to encounter approximately **1 UserWarning** and **8 ValueWarnings**.

## Expected Runtime

The approximate runtime of each script on the reference system is summarised below.

| Script | Expected Runtime |
|---------|---------:|
| `RELEASE.ConfigCheck` | < 1 s |
| `RELEASE.DataQuality` | < 5 min |
| `RELEASE.RunModels` | ~ 10 h |
| `RELEASE.RunSensitivity` | ~ 6.5 h |
| `RELEASE.FigureTable` | < 30 min |

The full reproduction workflow is expected to take approximately **20 hours** on the reference system.

Actual runtimes may vary depending on the hardware configuration, available memory, CPU core count, and system load.
